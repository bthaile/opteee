# Using AI as a Trader | Options Trading for Beginners

## Summary

Eric introduces Project No Code, a no-code quantitative research framework powered by AI (Claude, ChatGPT Codex) that enables traders without engineering backgrounds to build personal options research databases and conduct systematic backtesting. The session covers AI model selection, context window management, prompt engineering best practices, and the critical distinction between using AI as a force multiplier versus a replacement for trader judgment and research discipline.

## Key takeaways

- **Project No Code launch**: A free quick-start guide (20+ pages) and framework will release Sunday; Pro Plus members get Zoom workshops starting Monday to build alongside Eric [11:43]
- **AI is a force multiplier, not a replacement**: You remain the thinking, planning, and oversight layer; AI handles iteration speed and data processing [35:52]
- **Context window management is critical**: Monitor token usage (Claude 4.8 has 1M context); use markdown files (claude.md, SOP, roadmap) to preserve decision history across multiple chats and avoid context rot [37:24–45:51]
- **Chat models vs. coding agents differ**: Claude (chat) is good for planning and brainstorming; Claude Code (agent) operationalizes and executes; use both in sequence for best results [46:48–49:26]
- **Prompt engineering is the biggest lever**: Give AI a lens (e.g., "you are an options strategist"), ask it to brainstorm and punch holes in your ideas, and instruct it not to be agreeable [51:11–53:00]
- **Backtesting rigor is non-negotiable**: Watch "Basics of Backtesting" video before building; use bootstrapping, Monte Carlo, and randomized order checks to avoid overfitting [26:46–28:28]
- **Research requires domain expertise**: AI can't find edge for you; you must ask good questions (e.g., "When is variance risk premium richest?" not "Sell credit spreads") [33:52–36:50]
- **Hardware baseline**: 32 GB RAM minimum (96 GB DDR5 preferred); 4 TB SSD for hot storage; plan for redundant storage (Eric uses 3 copies of data) [59:20–01:08:12]
- **Smoke-test early**: Pull 25 tickers before 4,000; validate data shape and API behavior before scaling [19:52–20:14]
- **Variance risk premium example**: 68% of days (2007–May 2026) carry positive risk premium; 32% do not—selective entry beats perpetual selling [35:24–36:19]
