# My AI-Powered Trading System Build Process

## Summary

Eric walks through his workflow for building AI-assisted trading systems, emphasizing the iterative use of multiple LLM tools (Claude Code, Claude Opus, ChatGPT/Codex) to write, review, and refine code. The core discipline is continuous monitoring and validation to catch breaking changes early, with strategic trade-offs between thoroughness and development speed.

## Key takeaways

- **Multi-LLM review loop** [00:00–00:23]: Use Claude Code for initial generation, then feed output to Claude Opus or Codex for error-checking and refinement; this catches logic issues but adds latency.
- **Selective deep review** [00:23]: Apply rigorous multi-model review to high-stakes decisions (database architecture, convention design); skip it on lower-risk incremental changes to maintain velocity.
- **Continuous monitoring** [00:00]: Active oversight of system progress is non-negotiable; breaking changes compound quickly in trading systems.
- **Prompt iteration overhead** [00:23]: Back-and-forth prompting with multiple models slows development; balance thoroughness against time cost.
